# data/

Two files power the site. Both are generated — don't hand-edit.

## `tools.json` — the catalog (stable)

The vetted directory. Each tool carries only facts + our contribution:

```json
{
  "url": "https://whatsmyname.app",
  "name": "WhatsMyName Username Search",
  "category": "People & Identities",     // our canonical taxonomy (15 categories)
  "payload_type": "link",                 // link | bookmarklet | query-form | ...
  "source": "Cyber Detective (cipher387)",// primary attribution
  "sources": ["Cyber Detective (cipher387)", "OSINT Combine", "OSINT Research"],
  "source_count": 3                       // authority signal: listed in N collections
}
```

We keep the URL (a fact), our category, and provenance — never a source's own folder
structure. Regenerated by `tools/export_catalog.py` in the private working project.

## `status.json` — link health (refreshed)

Per-URL health, overwritten by the freshness workflow (`.github/workflows/freshness.yml`).
`rot_pct` = dead + suspected_dead + blocked_unknown as a share of checked links.

```json
{
  "generated_at": "...", "catalog_total": 2657, "checked": 2459, "rot_pct": 20.2,
  "by_state": {"alive": 1955, "blocked_unknown": 263, "dead": 116, "...": 0},
  "links": { "https://…": {"state": "alive", "final_status": 200, "checked_at": "…"} }
}
```

The committed copy is an initial snapshot from the last local run so the site shows health
on day one; the workflow keeps it current after launch.
